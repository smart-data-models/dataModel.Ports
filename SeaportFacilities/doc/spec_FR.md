Entité : SeaportFacilities  
==========================  
[Licence ouverte](https://github.com/smart-data-models//dataModel.Ports/blob/master/SeaportFacilities/LICENSE.md)  
Description globale : **Le modèle de données est destiné à fournir des informations sur les ports qui peuvent accueillir des bateaux de plaisance, des commerces ou le transport de passagers. Il permet de représenter les paramètres de chaque port, sa localisation, ses capacités d'amarrage et les services gratuits ou payants qui y sont associés, fournis directement par le port ou par des professionnels travaillant sur ou à proximité du port.**  

## Liste des biens  

- `address`: L'adresse postale.  - `alternateName`: Un autre nom pour cet article  - `areaServed`: La zone géographique où un service ou un article offert est fourni  - `authorizedPropulsion`: Type de propulsions autorisées à entrer dans le port. Une combinaison de moteur, voile, électrique, rame, nucléaire, lng, lpg, autre  - `boatSupplyingServices`: description des services complémentaires de fourniture du bateau offerts par les professionnels travaillant sur le port ou à proximité. Une combinaison de gardiennage, station de carburant, camion-citerne de carburant, camion-citerne d'eau potable, approvisionnement, chauffage à sec, chauffage à eau, réparation, expertise, passerelles, levage, grues, remorquage, pompage d'eaux usées, transport de bateaux, transfert de bateaux, autres  - `contactPoint`: https://schema.org/ContactPoint  - `contractingAuthority`: Nom du pouvoir adjudicateur  - `contractingCompany`: La société contractante responsable de la gestion du port.  - `currencyAccepted`: Une combinaison d'une liste de codes actifs définis dans le modèle [Norme ISO 4217](http://en.wikipedia.org/wiki/ISO_4217), [Crypto Currencies](https://en.wikipedia.org/wiki/List_of_cryptocurrencies) , [Exchange Trading System](https://en.wikipedia.org/wiki/Local_exchange_trading_system  - `dataProvider`: Une séquence de caractères identifiant le fournisseur de l'entité de données harmonisées.  - `dateCreated`: Horodatage de la création de l'entité. Il est généralement attribué par la plate-forme de stockage.  - `dateLastReported`: Un horodatage qui indique la dernière fois qu'un flux a transmis des données avec succès. La date et l'heure de ce référentiel au format ISO8601 UTC  - `dateModified`: Horodatage de la dernière modification de l'entité. Il est généralement attribué par la plate-forme de stockage.  - `description`: Une description de cet article  - `electricTransport`:  Liste des différents types de transport électrique proposés par la ville. Une combinaison de voiture électrique, bicyclette électrique, vélo électrique, scooter électrique  - `facilities`: Description des installations proposées sur le port. Combinaison de : toilettes, douches, laverie, téléphone, poubelles, bennes à ordures, conteneurs, tri sélectif des déchets, terminal électrique, terminal d'eau, réservation de chambre intérieure, wifi, autres  - `id`: Identifiant unique de l'entité  - `location`:   - `maxDraft`: Un tirant d'eau maximum autorisé pour accéder au port. Le code de l'unité (texte) est donné en utilisant les [codes communs du CEFACT-ONU] (http://wiki.goodrelations-vocabulary.org/Documentation/UN/CEFACT_Common_Codes) . Par exemple, **MTR** représente Meter  - `maxLength`: Une longueur maximale autorisée pour accéder au port. Le code de l'unité (texte) est donné en utilisant les [codes communs du CEFACT-ONU] (http://wiki.goodrelations-vocabulary.org/Documentation/UN/CEFACT_Common_Codes) . Par exemple, **MTR** représente Meter  - `maxTonnage`: Tonnage maximum autorisé pour accéder au port. Le code de l'unité (texte) est donné en utilisant les [codes communs du CEFACT-ONU] (http://wiki.goodrelations-vocabulary.org/Documentation/UN/CEFACT_Common_Codes) . Par exemple, **TNE** représente la tonne métrique.  - `maxWidth`: Une largeur maximale autorisée pour accéder au port. Le code de l'unité (texte) est donné en utilisant les [codes communs du CEFACT-ONU] (http://wiki.goodrelations-vocabulary.org/Documentation/UN/CEFACT_Common_Codes) . Le code de l'unité (texte) est donné en utilisant les [codes communs du CEFACT-ONU] (http://wiki.goodrelations-vocabulary.org/Documentation/UN/CEFACT_Common_Codes). Par exemple, **MTR** représente Meter  - `name`: Le nom de cet article.  - `nearbyServices`: Description des services supplémentaires sur la zone géographique du port ou à proximité. Une combinaison de : bureau de tourisme, magasins d'équipement, agence de voyage, bureau de change, bureau médical, pharmacie, épicerie, restaurants, presse, bar, magasins, excursions en mer, cityTour, excursions touristiques, autres  - `numberOfPlace`: Nombre total de places dans le port.  - `openingHoursSpecification`: Une valeur structurée fournissant des informations sur les heures d'ouverture d'un lieu ou d'un certain service à l'intérieur d'un lieu.  - `owner`: Une liste contenant une séquence de caractères codés en JSON faisant référence aux Ids uniques du ou des propriétaires  - `paymentAccepted`: Paiement accepté. Une combinaison de : location de bateaux, vente de bateaux, location de jetSki, vente de jetSki, location de voitures, location de voitures de luxe, location de van, location de vélos, location de scooters, caddie, paletteTransport, autre  - `routeType`: Liste des différents modes de transport urbain (métro, bus, tramway, ...) selon la norme GFTS [STOP](https://developers.google.com/transit/gtfs/reference/#stopstxt). Une combinaison de valeurs composée uniquement de l'attribut "description" tram(0), metro(1), train(2), bus(3), ferry(4), cableTram(5), cableCar(6), funiculaire(7), trolleybus(11), monorail(12)  - `seeAlso`: liste d'uri pointant vers des ressources supplémentaires sur le sujet  - `source`: Une séquence de caractères donnant comme URL la source originale des données de l'entité. Il est recommandé d'utiliser le nom de domaine complet du fournisseur de la source, ou l'URL de l'objet source.  - `transportServices`: Description des services fournis pour le transport spécialisé et les services de navette. Une combinaison de : arches, navettes vers l'aéroport, navettes vers le chemin de fer, navettes internes, taxis, héliport, autres  - `type`: Il faut que ce soit SeaPort  - `typeOfPort`: Un type de port. Une combinaison de : marina, marchandises, croisière, ferry, passagers, yachting, pêche, militaire, fluvial, autre  - `webSite`: Lien vers le site officiel du port pour plus d'informations.
